# ElevateLabs-Task6
This task focuses on using subqueries inside SELECT, WHERE, and FROM clauses to perform advanced data retrieval operations in SQL. Two tables — Customers and Orders — were used to demonstrate different types of subquery logic.
Subqueries and Nested Queries

This task focuses on writing subqueries inside SELECT, WHERE, and FROM clauses to perform advanced data retrieval operations.

Objective

Learn scalar and correlated subqueries

Use subqueries inside IN, EXISTS, and =

Understand nested logic and multi-level queries

What Was Done

Created sample Customers and Orders tables

Inserted realistic data

Performed:

Scalar subqueries (SELECT)

Subqueries with IN

Subqueries with EXISTS

Subqueries with =

Inline view (subquery in FROM)

Correlated subqueries

Outcome

Ability to write nested SQL logic

Understanding of how subqueries work inside different clauses

Strong foundation in advanced SQL concepts
